## Огляд

Агент складається з пайплайна (`src/main.py`) та CustomTkinter GUI (`src/ui.py`). Він:

1. Підключається до Figma API, витягує фрейм за `file key` / `node id`.
2. Нормалізує UI-компоненти у JSON (`normalized.json`).
3. Формує промпт та викликає OpenAI або Anthropic для генерації `section.html` та `styles.css`.
4. Збирає preview, логи, трейси, assets.
5. Дозволяє запускати все з CLI або зручного GUI з повною прокруткою, темами, анімаціями та асинхронним виконанням.

## Можливості

- Перетворення будь-якого Figma Frame у готовий HTML/CSS.
- Dry-run без LLM (лише нормалізація).
- Підтримка `normalized.json` як вхідних даних без звернення до Figma.
- Запис трейсу для аудиту чи fine-tune датасету.
- Експорт растрових/векторних ресурсів у `output/assets/`.
- Адаптивний GUI з темною/світлою темою, кольоровими логами, перемикачами, прогресбаром.
- Упакування в `.exe` без додаткових консольних вікон і без рекурсивних підпроцесів.

## Архітектура та файли

| Файл/каталог | Призначення |
| --- | --- |
| `src/main.py` | CLI-вхідна точка, парсинг аргументів, orchestration пайплайна, `run_with_args` для in-process запуску в GUI. |
| `src/ui.py` | CustomTkinter GUI: поля вводу, перемикачі, лог, асинхронний раннер, прокрутка, інтеграція з `src.main`. |
| `src/traces.py` | Утиліти формування трейсу та структурування файлів. |
| `output/` | Основний каталог результатів (HTML, CSS, preview, normalized JSON, assets). |
| `traces/` | Архівовані прогони з метаданими, якщо ввімкнено `--record-trace`. |
| `figma-agent.spec` | PyInstaller-конфіг для збирання GUI в `.exe`. |

Ланцюжок виконання: **Fetch → Normalize → Prompt (LLM) → Post-process → Trace**.

## Вимоги

- Python 3.10+ (перевірено на Windows 11).
- Доступний Figma Personal Access Token.
- OpenAI або Anthropic API key (залежно від `LLM_PROVIDER`).
- pip + можливість створювати virtualenv.

## Налаштування середовища

```bash
python -m venv .venv
.venv\Scripts\activate        # або source .venv/bin/activate на Unix
pip install -r requirements.txt

copy .env.example .env         # створюємо файл змінних середовища
```

Далі заповніть `.env` згідно таблички нижче.

## Ключові змінні `.env`

| Змінна | Опис |
| --- | --- |
| `FIGMA_TOKEN` | Обовʼязковий. Особистий токен Figma. |
| `FIGMA_FILE_KEY`, `FIGMA_NODE_ID` | Можна задати дефолтні значення, GUI підставляє їх автоматично. |
| `LLM_PROVIDER` | `openai` або `anthropic`. Визначає, який SDK використається. |
| `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` | Ключ до відповідного провайдера. |
| `OPENAI_MODEL`, `ANTHROPIC_MODEL` | Опційні override назв моделей (наприклад, `gpt-4o-mini`). |
| `OUTPUT_DIR` | Директорія, куди падатимуть HTML/CSS/preview. За замовчуванням `output`. |
| `TRACE_DIR` | Директорія для трейсу (типово `traces`). |
| `FIGMA_MAX_RETRIES`, `FIGMA_BACKOFF_BASE` | Контроль експоненційного backoff на випадок 429/5xx від API. |

GUI також читає ці значення при старті, тож можна не дублювати їх вручну у формах.

## Запуск через CLI

```bash
# Повний прогін з Figma, LLM та збереженням у output/
python -m src.main --file-key YOUR_FILE --node-id 0:1 --output-dir output

# Використати готовий normalized JSON, щоби пропустити Figma
python -m src.main --normalized-input output/normalized.json --dry-run

# Записати трейси і зберегти assets
python -m src.main --file-key ... --record-trace --trace-dir traces --export-assets
```

### Корисні прапорці

- `--dry-run` – нормалізувати і зупинитися (без виклику LLM).
- `--normalized-input path` – читати вже нормалізований JSON, не робити запит до Figma.
- `--record-trace` – архівувати повний прогін у `trace-dir/timestamp/`.
- `--export-assets` – викачати зображення та зберегти в `output/assets/`.

За відсутності `--node-id` береться перший frame у файлі.

## Графічний інтерфейс

```bash
python -m src.ui             # режим розробки
dist/figma-agent.exe         # зібраний варіант
```

Особливості GUI:

1. Темна/світла тема, скруглені картки, hover-анімації, плавні кнопки.
2. Великі поля вводу з підказками та кнопкою вибору `normalized.json`.
3. Тумблери Dry run / Record trace / Export assets з чіткою індикацією.
4. Асинхронний запуск пайплайна з прогресбаром і блокуванням кнопки «Запустити».
5. Кольоровий лог (info/success/warning/error) зі смутою стану.
6. Повністю прокручуваний інтерфейс (усі блоки у `CTkScrollableFrame`).
7. У зібраному `.exe` GUI викликає `src.main` напряму (in-process) без рекурсії та без консольного вікна.

### Сценарій роботи в GUI

1. Відкрити `python -m src.ui`.
2. Ввести/перевірити токен, File Key, Node ID або вибрати готовий `normalized.json`.
3. Налаштувати директорії `Output` і `Trace`, обрати потрібні тумблери.
4. Натиснути «Запустити» й читати лог виконання; помилки підсвічуються червоним.
5. Після завершення кнопка знову активна, а результати лежать у `output/`.

## Структура вихідних даних

За замовчуванням у `output/` зʼявляються:

- `normalized.json` — нормалізовані вузли Figma.
- `section.html` — HTML-фрагмент секції.
- `styles.css` — стилі, які узгоджені з HTML.
- `preview.html` — автономна сторінка для швидкого перегляду (відкрийте в браузері).
- `assets/` — зображення та іконки (якщо ввімкнено експорт).

## Трейси, логування й асети

- Логи CLI друкуються в консоль; GUI має вікно лога з автоскролом і підсвічуванням.
- Прапорець `--record-trace` створює структуру `traces/<ISO-час>/` з:
  - `normalized.json`, HTML/CSS, preview;
  - параметрами запуску;
  - сирими відповідями API;
  - asset-файлами.
- Ці трейси зручно використовувати як датасет для донавчання чи розбору помилок.

## Збирання `.exe` (Windows)

Використовуйте PyInstaller (або готовий `figma-agent.spec`):

```bash
pip install pyinstaller
pyinstaller --noconsole --onefile --name figma-agent src/ui.py
```

Рекомендації:

- `--noconsole` вимикає додаткове консольне вікно.
- `.env` має бути доступним (або покладіть його поруч із `.exe`, або підготуйте системні змінні).
- У зібраному режимі не потрібно передавати аргументи — GUI працює як самостійний застосунок.

## Типові сценарії

1. **Швидке превʼю секції** — CLI з `--file-key` + `--node-id` генерує `preview.html` за кілька секунд.
2. **Дебаг промптів** — dry-run + ручне редагування `normalized.json` → повторний прогін із `--normalized-input`.
3. **Збір датасету** — для кожного макета запуск з `--record-trace --export-assets` і архівація трейсу.
4. **Робота дизайнерів** — збірка `.exe`, де все робиться через GUI без командного рядка.

## FAQ / Траблшутинг

**GUI не бачить токен** — перевірте `.env` у корені проєкту й перезапустіть застосунок.

**Отримую 401/403 від Figma** — токен недійсний або не має доступу до файлу. Переконайтеся, що ви редактор документа.

**LLM відповідає з rate limit** — використайте іншу модель/обліковку або зменшіть паралельні запити; в CLI можна додати retry-логіку.

**CLI зупинився на нормалізації** — ви могли передати `--dry-run` або не вказали жодного джерела (ані `--file-key`, ані `--normalized-input`).

**Зібраний `.exe` відкриває консоль** — збирайте з `--noconsole` та переконайтеся, що використовується актуальний `src/ui.py` з in-process запуском.

**Прокрутка не працює** — усі секції знаходяться у `CTkScrollableFrame`. Якщо скролбару не видно, просто використовуйте колесо миші/жест; коли вміст перевищує висоту, смуга зʼявиться автоматично.

**Де результати?** — шукайте у `OUTPUT_DIR` (типово `output/`): `section.html`, `styles.css`, `preview.html`, `normalized.json`, `assets/`.
